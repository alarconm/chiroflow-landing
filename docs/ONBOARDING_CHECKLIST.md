# ChiroFlow Onboarding Checklist
## From Handshake to Automation in 30 Days

---

## Phase 0: Pre-Kickoff (Before Day 1)

### Legal & Compliance
- [ ] LOI signed by both parties
- [ ] BAA (Business Associate Agreement) executed
- [ ] NDA in place (if separate from LOI)
- [ ] Confirm HIPAA compliance requirements understood

### Access Setup
- [ ] Secure document sharing established (Google Drive folder with restricted access)
- [ ] Communication channel set up (Slack channel, or dedicated email thread)
- [ ] Calendar invites for kickoff and weekly check-ins scheduled

### Initial Documents Requested
- [ ] Request list of all current software systems
- [ ] Request sample SOPs (2-3 examples)
- [ ] Request organizational chart / staff list
- [ ] Request sample reports they run regularly

---

## Phase 1: Discovery (Days 1-7)

### Day 1: Kickoff Call (90 minutes)

**Attendees:** Mike, Kent, [Office Manager if applicable]

**Agenda:**
- [ ] Introductions and partnership overview
- [ ] Walk through Discovery Questionnaire
- [ ] Identify top 3 automation priorities
- [ ] Schedule system access and SOP handoff
- [ ] Set expectations for communication cadence

**Deliverables:**
- [ ] Completed Discovery Questionnaire
- [ ] Prioritized automation list
- [ ] Next steps documented and shared

### Days 2-5: Document Collection

- [ ] All SOP documents received
- [ ] Access to practice management system (read-only/sandbox)
- [ ] Sample EOBs collected (5-10 examples, PHI redacted or test data)
- [ ] Sample AR aging report
- [ ] Sample verification workflow
- [ ] List of top 10 payers with portal access

### Days 5-7: Process Mapping

- [ ] Payment posting workflow documented (SOP template)
- [ ] AR management workflow documented
- [ ] Insurance verification workflow documented
- [ ] Key decision points identified
- [ ] Exception handling documented
- [ ] Time estimates validated with Kent's team

**Milestone: Discovery Complete**
- [ ] All priority workflows mapped
- [ ] Technical requirements identified
- [ ] Automation approach approved by Kent

---

## Phase 2: Setup (Days 8-14)

### Technical Environment

- [ ] HIPAA-compliant hosting provisioned
- [ ] Database set up with encryption
- [ ] Secure credential storage configured
- [ ] Browser automation environment ready
- [ ] Monitoring and logging enabled

### System Integration

- [ ] Practice management system access tested
- [ ] EHR login working via automation
- [ ] Payer portal access tested (top 5 payers)
- [ ] Email integration configured
- [ ] Document storage connected

### Agent Configuration

- [ ] Billing Agent configured with Kent's rules
- [ ] AR Agent configured with follow-up logic
- [ ] Verify Agent configured with payer list
- [ ] Reports Agent configured with report templates
- [ ] Approval workflows set up

**Milestone: Environment Ready**
- [ ] All integrations tested
- [ ] Agents configured
- [ ] Ready for first automation test

---

## Phase 3: Build & Test (Days 15-25)

### Week 3: First Automation - Payment Posting

**Day 15-17: Build**
- [ ] EOB parsing automation built
- [ ] Patient/claim matching logic implemented
- [ ] Payment posting workflow completed
- [ ] Adjustment handling configured

**Day 18-19: Test**
- [ ] Test with 10 sample EOBs
- [ ] Accuracy measured (target: >95%)
- [ ] Edge cases identified and handled
- [ ] Kent reviews results

**Day 20-21: Iterate**
- [ ] Fixes applied based on testing
- [ ] Re-test with 20 more samples
- [ ] Accuracy validated
- [ ] Approval thresholds set

### Week 4: Second Automation - AR Management

**Day 22-23: Build**
- [ ] Aging report automation built
- [ ] Follow-up logic implemented
- [ ] Payer portal checking automated

**Day 24-25: Test**
- [ ] Test with current AR data
- [ ] Review prioritization logic
- [ ] Validate follow-up actions
- [ ] Kent approves approach

**Milestone: Core Automations Working**
- [ ] Payment posting: >95% accuracy
- [ ] AR management: Logic approved
- [ ] Ready for production pilot

---

## Phase 4: Production Pilot (Days 26-30)

### Go-Live Preparation

- [ ] Production credentials configured
- [ ] Backup/rollback plan documented
- [ ] Escalation contacts identified
- [ ] Daily monitoring dashboard ready

### Pilot Launch

- [ ] Start with 1 day of EOBs (supervised)
- [ ] Human reviews all automated postings
- [ ] Issues documented and fixed
- [ ] Expand to full day 2

### Pilot Monitoring

- [ ] Daily accuracy tracking
- [ ] Time savings measured
- [ ] Exception rate monitored
- [ ] Staff feedback collected

**Milestone: Pilot Complete**
- [ ] 5 days of successful operation
- [ ] Accuracy maintained >95%
- [ ] Time savings validated
- [ ] Kent gives go-ahead for full production

---

## Phase 5: Full Production (Day 30+)

### Handoff to Operations

- [ ] Reduce human oversight (spot-check only)
- [ ] Automated reporting enabled
- [ ] Exception handling refined
- [ ] Staff trained on working with agents

### Ongoing Optimization

- [ ] Weekly review calls (first month)
- [ ] Bi-weekly calls (months 2-3)
- [ ] Monthly calls (ongoing)
- [ ] Continuous improvement based on data

### Expansion Planning

- [ ] Identify next automation candidates
- [ ] Prioritize based on ROI
- [ ] Schedule additional implementations

---

## Success Criteria

### Quantitative

| Metric | Target | Measurement |
|--------|--------|-------------|
| Payment posting accuracy | >95% | Weekly audit |
| Time savings | >30 hrs/week | Before/after comparison |
| Exception rate | <5% | Automated tracking |
| Processing time | <24 hrs | SLA monitoring |

### Qualitative

- [ ] Kent satisfied with automation quality
- [ ] Staff comfortable working with agents
- [ ] No compliance concerns
- [ ] Willing to provide testimonial/referral

---

## Communication Schedule

| Week | Cadence | Format | Duration |
|------|---------|--------|----------|
| 1 | Daily | Slack/email | As needed |
| 2-4 | Every other day | Quick call | 15 min |
| 5-8 | Weekly | Video call | 30 min |
| 9+ | Bi-weekly | Video call | 30 min |

**Escalation Path:**
1. Slack message (response: <4 hours)
2. Email (response: <24 hours)
3. Phone call (urgent issues only)

---

## Risk Mitigation

| Risk | Mitigation | Owner |
|------|------------|-------|
| System access issues | Have backup manual process | Kent's team |
| Accuracy problems | Human review layer, quick iteration | ChiroFlow |
| Staff resistance | Training, show time savings | Both |
| HIPAA concerns | Documentation, audit trail | ChiroFlow |
| Integration failures | Browser fallback, manual override | ChiroFlow |
