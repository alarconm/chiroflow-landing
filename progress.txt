## US-218: Virtual waiting room
**Status:** COMPLETED
**Date:** 2026-01-22

### What was implemented:

1. **telehealth.joinWaitingRoom tRPC procedure**:
   - Patient joins the virtual waiting room before their telehealth visit
   - Records device type, browser info, and connection type
   - Creates VirtualWaitingRoom record linked to session
   - Updates session status to WAITING
   - Supports rejoining if patient previously left

2. **telehealth.getWaitingRoom tRPC procedure**:
   - Provider view: Lists all patients waiting for their appointments
   - Patient view: Gets specific waiting room details when sessionId provided
   - Includes patient demographics, appointment type
   - Calculates and returns current wait time in seconds
   - Polls every 10 seconds for provider dashboard

3. **telehealth.admitPatient tRPC procedure**:
   - Provider admits patient from waiting room to session
   - Verifies only assigned provider can admit
   - Calculates total wait time
   - Updates waiting room status to ADMITTED
   - Updates session status to IN_PROGRESS
   - Records actual session start time
   - Audit logging for compliance

4. **telehealth.leaveWaitingRoom tRPC procedure**:
   - Patient can voluntarily leave waiting room
   - Records leave time and total wait seconds
   - Updates status to LEFT

5. **telehealth.updateTechnicalCheck tRPC procedure**:
   - Updates pre-visit checklist: camera, microphone, connection
   - Allows incremental updates as each check completes

6. **telehealth.submitPreVisitQuestionnaire tRPC procedure**:
   - Submits patient responses to pre-visit questions
   - Stores as JSON for provider review
   - Marks questionnaire as completed

7. **telehealth.getWaitingRoomStatus tRPC procedure**:
   - Patient-facing status query
   - Returns formatted wait time (MM:SS)
   - Indicates if provider is ready (ADMITTED or IN_PROGRESS)
   - Includes pre-check status and questionnaire completion
   - Polls every 5 seconds for real-time updates

8. **VirtualWaitingRoom component** (`app/src/components/telehealth/VirtualWaitingRoom.tsx`):
   - Patient-facing waiting room UI
   - Auto-joins waiting room on mount
   - Real-time wait time display
   - Technical pre-checks (camera, mic, connection) with visual feedback
   - Pre-visit questionnaire form
   - Provider ready notification with alert
   - Leave waiting room option
   - Responsive design with loading states

9. **ProviderWaitingRoomDashboard component** (`app/src/components/telehealth/ProviderWaitingRoomDashboard.tsx`):
   - Provider-facing dashboard showing all waiting patients
   - Patient avatars and names
   - Current wait times per patient
   - Technical check status icons (green/red)
   - Pre-visit questionnaire view dialog
   - Admit button to start session
   - Waiting room statistics (total waiting, avg wait time, ready count)
   - Auto-refresh every 10 seconds

**Date:** 2026-01-22

   - Sends consent form request to patient
   - Creates pending consent record
   - Supports resending existing pending consents
   - Custom expiration days (1-730, default 365)
   - Optional notification flag
   - Custom consent text override support

2. **telehealth.checkConsent tRPC procedure**:
   - Checks consent status for a patient
   - Supports multiple consent types in single query
   - Returns detailed validity information per consent type
   - Calculates days until expiration
   - Determines if renewal is needed (within 30 days)
   - Checks state-specific requirements
   - Returns actionable recommendations

3. **telehealth.signConsent tRPC procedure**:
   - Digital signature capture support
   - Supports typed and drawn signatures
   - Base64 encoded signature data storage
   - Acknowledgment verification required
   - IP address tracking for audit
   - Full audit trail with metadata

4. **telehealth.getConsentDetails tRPC procedure**:
   - Returns full consent text and details
   - Marks consent as viewed (for audit)
   - Includes patient information
   - Secure - doesn't expose raw signature data

5. **telehealth.renewConsent tRPC procedure**:
   - Creates new consent based on expired one
   - Preserves old consent for audit trail
   - Marks old consent as EXPIRED
   - Configurable new expiration period

6. **telehealth.revokeConsent tRPC procedure**:
   - Allows consent revocation
   - Requires revocation reason
   - Full audit trail

7. **telehealth.listPatientConsents tRPC procedure**:
   - Lists all consents for a patient

8. **telehealth.validateSessionConsent tRPC procedure**:
